# Phase 63C Secret Rotation and History-Purge Ceremony

## Preconditions

- Approved change window and rollback owner.
- Repository write freeze announced.
- Mirror clone created outside the developer working tree.
- Six replacement credentials generated by SecOps and stored directly in Vault.
- Phase 61 supply-chain attestation copied from its template and stored outside Git.

## Execution sequence

1. Rotate `POSTGRES_PASSWORD`, `REPLICATION_PASSWORD`, `DB_APP_PASSWORD`, `FLYWAY_PASSWORD`, `ARCHIVE_POSTGRES_PASSWORD` and `MINIO_ROOT_PASSWORD`.
2. Validate new credentials from UAT, then disable the old values.
3. Run `security/scripts/purge-sensitive-history.sh --execute` on the approved mirror clone.
4. Force-push only after Repo Coordinator approval.
5. Invalidate CI caches and service-account tokens; require all developers to re-clone.
6. Run strict repository hygiene and all-branch/tag secret scanning.
7. Sign the Phase 61 supply-chain attestation. Do not include credential values.
8. Run Phase 63C in full mode to verify the signed evidence.

## Rollback

Do not restore old compromised credentials. If the new deployment fails, issue a new credential generation, update Vault and redeploy the last approved image digest.
