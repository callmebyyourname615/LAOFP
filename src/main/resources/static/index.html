Demo Notes

จุดที่ควรเล่าประกอบเวลาพรีเซนต์ switching demo

  • เริ่มด้วยการอธิบายว่า request เข้ามาผ่าน channel API พร้อม `X-Channel-Id` และ `Idempotency-Key`
  • หลัง create สำเร็จ ระบบจะตอบ `transferRef` กลับก่อน แล้ว worker จะ dispatch ต่อแบบ asynchronous
  • โชว์การเปลี่ยนสถานะ `RECEIVED -> PROCESSING -> SUCCESS/PENDING/FAILED` เพื่อเล่า event-driven flow
  • ถ้าต้องการ demo duplicate protection ให้กดส่งซ้ำโดยไม่เปลี่ยน `Idempotency-Key`

Portal นี้เรียก API จาก origin เดียวกับ Spring Boot app จึงเหมาะสำหรับ demo แบบ local environment และขยายต่อไปเป็น portal ของ microservices ได้ภายหลัง
